# 小程序页面设计规范 (Mini Program Design Spec)

> 基于 OpenSpec 理念编写，旨在通过明确的规范定义，确保 AI 生成的小程序页面符合品牌调性、用户体验标准及技术实现要求。

## 一、 设计原则 (Design Principles)

*   **品牌一致性 (Consistency)**：遵循品牌 VI（色值、字体、圆角），传递统一的品牌感知。
*   **轻量高效 (Lightweight)**：去繁就简，聚焦核心任务路径，避免过度设计。
*   **拇指友好 (Thumb-friendly)**：核心操作区位于屏幕下半部分，便于单手操作。
*   **沉浸体验 (Immersive)**：利用全屏背景、微动效提升用户留存。

---

## 二、 基础样式规范 (Visual Style)

### 1. 调色板 (Color Palette)
*   **主色 (Primary)**：`#FF5A5F` (品牌红，用于按钮、核心图标)
*   **辅助色 (Secondary)**：`#00D1C1` (成功/安全)，`#FFB400` (警告/会员)
*   **中性色 (Neutrals)**：
    *   `#1A1A1A` (主标题)
    *   `#666666` (正文)
    *   `#999999` (辅助文案)
    *   `#F5F5F5` (背景底色)
    *   `#E0E0E0` (分割线)

### 2. 字体排印 (Typography)
*   **字体家族**：`PingFang SC` (iOS), `Roboto` (Android), `sans-serif`
*   **字号层级**：
    *   **大标题 (H1)**：`20px` (加粗，用于导航栏、核心数字)
    *   **标题 (H2)**：`16px` (加粗，用于模块标题)
    *   **正文 (Body)**：`14px` (常规，用于列表、详情)
    *   **辅助 (Caption)**：`12px` (常规，用于标签、时间)
    *   **最小 (Small)**：`10px` (仅用于底部 Tab 标签)

### 3. 布局与间距 (Layout & Spacing)
*   **页面边距**：左右统一 `16px` (32rpx)。
*   **卡片圆角**：统一 `12px` (24rpx)。
*   **按钮圆角**：
    *   大按钮（主操作）：`44px` (高度 88rpx，全圆角)
    *   小按钮（辅助）：`4px`
*   **间距阶梯**：`8px`, `16px`, `24px`, `32px`。

## 三、 组件设计规范 (Component Specs)

### 1. 导航栏 (Navigation Bar)
*   **风格**：
    *   **沉浸式**：背景透明，白色文字（用于个人中心、活动页）。
    *   **标准式**：白色背景，黑色文字，带底部投影（用于列表页、表单页）。
*   **返回按钮**：统一使用黑色/白色胶囊风格，保持与胶囊按钮对齐。

### 2. 底部操作栏 (Bottom Action Bar)
*   **吸底布局**：固定在底部，`z-index: 100`。
*   **安全区适配**：必须包含 `padding-bottom: env(safe-area-inset-bottom)`。
*   **结构**：
    *   左侧：辅助操作（客服、购物车），图标 + 文字。
    *   右侧：主操作按钮（立即购买），撑满剩余空间。

### 3. 列表卡片 (List Card)
*   **商品卡片**：
    *   **布局**：左图右文 / 双列瀑布流。
    *   **图片**：`aspectFill` 裁剪，圆角 `8px`。
    *   **价格**：主色高亮，整数部分字号放大。
    *   **标签**：圆角 `2px`，字号 `10px`，背景色为主色透明度 `10%`。
*   **功能列表**：
    *   **高度**：`56px`。
    *   **结构**：Icon + 标题 + (说明文字) + 右箭头。
    *   **分割线**：非最后一行显示分割线，左缩进 `16px`。

### 4. 弹窗与提示 (Modals & Toasts)
*   **Toast**：轻量提示，1.5s 后消失，文字居中，半透明黑底。
*   **Dialog**：重要决策，居中弹窗，带遮罩。
    *   **标题**：居中加粗。
    *   **按钮**：双按钮时，“取消”在左（灰字），“确定”在右（主色字）。
*   **ActionSheet**：多选项操作，从底部滑出。

## 四、 交互体验规范 (Interaction & UX)

### 1. 加载与空状态
*   **骨架屏 (Skeleton)**：首屏加载必须使用骨架屏，禁止长时间白屏。
*   **下拉刷新**：使用小程序原生 `enablePullDownRefresh`，Loading 样式统一。
*   **上拉加载**：列表底部显示“加载中...”或“没有更多了”。
*   **空状态**：
    *   图片：品牌定制插画，占屏宽 50%。
    *   文案：友好提示（如“购物车还是空的”）。
    *   按钮：引导操作（如“去逛逛”）。

### 2. 动画效果
*   **页面转场**：使用小程序原生推入效果。
*   **按压态 (Hover)**：
    *   按钮/列表项点击时，透明度变为 `0.8` 或背景变灰 (`#F5F5F5`)。
    *   `hover-class="hover-opacity"`。

## 五、 技术实现指引 (For AI)

为了确保生成的代码符合规范，请遵循以下实现细节：

### 1. 样式单位 (Units)
*   统一使用 `rpx` 作为布局单位，以适配不同屏幕宽度（基准 750rpx）。
*   字体大小可使用 `px` 或 `rpx`（建议 `rpx` 以随屏幕缩放）。

### 2. CSS 变量 (CSS Variables)
在 `app.wxss` 中定义全局变量，页面中直接引用：
```css
page {
  --color-primary: #FF5A5F;
  --color-text-main: #1A1A1A;
  --spacing-base: 32rpx;
  --radius-card: 24rpx;
}

.button {
  background-color: var(--color-primary);
}
```

### 3. 图片资源
*   优先使用 SVG 图标（Base64 或网络地址）。
*   大图必须使用 CDN 地址，禁止打包进小程序包。

### 4. 适配处理
*   **刘海屏**：Navbar 和 Tabbar 需处理 `safe-area-inset-top` 和 `safe-area-inset-bottom`。
*   **暗黑模式**：暂不强制适配，但建议预留 `media query` 结构。

## 六、 页面模板示例 (Page Templates)

### 1. 首页 (Home)
*   **结构**：搜索栏 (Sticky) -> Banner 轮播 -> 金刚区 (Grid) -> 营销位 -> 瀑布流商品。

### 2. 商品详情 (Product Detail)
*   **结构**：主图轮播 (1:1) -> 价格标题区 -> 规格选择 -> 详情图 -> 底部吸底栏。

### 3. 个人中心 (Profile)
*   **结构**：用户信息卡片 (背景图) -> 订单状态栏 (Flex) -> 常用功能列表。
