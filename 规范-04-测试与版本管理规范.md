# 测试与版本管理规范

## 一、 测试规范
- **技术栈**：JUnit 5 + Spring Boot Test + Mockito。
- **要求**：
    - 核心业务 Service 必须编写单元测试或集成测试。
    - 使用 `@MockBean` 隔离外部依赖（如短信、支付）。
    - 测试方法命名：`should_xxx_when_xxx`。
    - 核心业务模块行覆盖率建议 > 60%。

## 二、 Git 提交规范
- **Commit Message** 遵循 Conventional Commits：
    - `feat`: 新功能
    - `fix`: 修复 Bug
    - `docs`: 文档变更
    - `style`: 代码格式调整
    - `refactor`: 代码重构
    - `chore`: 工具链/构建变动
- **分支管理**：
    - `main`: 生产稳定分支。
    - `develop`: 开发主分支。
    - `feature/*`: 功能开发分支。
