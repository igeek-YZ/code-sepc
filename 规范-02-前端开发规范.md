# 前端开发规范

## 一、 技术栈要求 (前端)
- 前端框架：Vue 3 + Element Plus 
- 状态管理：Pinia
- 构建工具：Vite
- 样式：Sass / Less

## 二、 目录结构规范
- `src/views`: 页面视图组件
- `src/components`: 全局通用组件
- `src/hooks`: 组合式 API (Composables)
- `src/store`: 状态管理
- `src/api`: 后端接口封装

## 三、 命名规范
- **组件文件**：使用大驼峰命名 (PascalCase)，如 `UserProfile.vue`。
- **目录命名**：使用小写短横线 (kebab-case)，如 `user-profile`。
- **变量/函数**：使用小驼峰 (camelCase)。

## 四、 代码实现规范
- **网络请求**：统一封装 Axios，拦截器处理 `Result<T>`。
- **API 封装**：API 模块与后端 Controller 对应，方法名保持语义一致。
- **状态管理**：业务逻辑尽量抽离到 hooks 或 store 中。
