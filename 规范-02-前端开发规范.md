# 前端开发规范

## 一、 技术栈要求 (前端)
- 前端框架：Vue 3 + Element Plus 
- 状态管理：Pinia
- 构建工具：Vite
- 样式：Sass / Less
- 路由：Vue Router

## 二、 目录结构规范

### 1. 项目目录结构
```text
src/
├── api/            # 后端接口封装，按模块划分子目录
├── assets/         # 静态资源（图片、全局样式）
├── components/     # 全局通用业务组件
├── hooks/          # 组合式 API (Composables)
├── layout/         # 页面布局组件
├── router/         # 路由配置
├── store/          # 状态管理 (Pinia)
├── utils/          # 工具函数封装
└── views/          # 页面视图，按模块划分子目录
    ├── login/      # 登录模块
    ├── order/      # 订单模块
    └── product/    # 商品模块
```

## 三、 命名规范

### 1. 文件命名规则
- **组件文件**：使用大驼峰命名 (PascalCase)，如 `OrderList.vue`。
- **目录命名**：使用小写短横线 (kebab-case)，如 `order-detail`。
- **API 文件**：与后端模块对应，如 `order.ts`, `user.ts`。

### 2. 代码编写规范
- **变量/函数**：使用小驼峰 (camelCase)。
- **常量**：全大写下划线 (UPPER_SNAKE_CASE)。
- **Props**：在模板中使用 kebab-case，在 JS 中使用 camelCase。

## 四、 代码实现规范

### 1. 网络请求封装
- 统一封装 Axios，拦截器必须处理 `Result<T>` 响应结构。
- 错误处理：非 200 状态码统一由拦截器弹出错误提示。
- 示例代码：
```typescript
// src/api/request.ts
service.interceptors.response.use(
  (response) => {
    const res = response.data;
    if (res.code !== '200') {
      ElMessage.error(res.msg || 'Error');
      return Promise.reject(new Error(res.msg || 'Error'));
    }
    return res.data;
  }
);
```

### 2. API 封装规范
- 每个接口必须包含详细的注释。
- 方法名应与后端 Controller 语义保持一致。
```typescript
// src/api/order.ts
export const getOrderById = (id: number) => {
  return request({
    url: `/api/order/${id}`,
    method: 'get'
  });
};
```

### 3. 状态管理规范
- 复杂的业务逻辑应尽量抽离到 hooks 或 Pinia store 中。
- Store 命名：`use{Module}Store`，如 `useUserStore`。
